**문서 목적:** PM이 작성하는 정책서를 기반으로 AI가 UI 프로토타입을 생성할 수 있도록 화면 정책을 정의한다.

> **데이터 기준**
>
> - 이 화면은 **온라인 판매 성과 분석** 중심으로 구성된다.

>

> - **MD 팀**: 소진율, 소진예상일, 발주 지표, 재고/입고 현황, 컬렉션별 판매 성과

# 1. 화면 이름

**Sales Dashboard**

---

# 2. 이 화면의 목적

**이커머스·MD 조직**이 온라인 판매 성과를 한 화면에서 확인하는 원페이지 대시보드다.

이 화면은 집계된 판매 성과(뭐가 얼마나 팔렸어?)와 그 원인(왜 그렇게 됐어?)을 함께 다룬다.

**핵심 가치**
- 이커머스 팀 — 전환 퍼널에서 이탈 단계 파악, 채널별 광고 효율 판단, 고객 재구매 흐름 모니터링
- MD 팀 — 재고 소진 속도와 입고 일정 파악, 다음 발주/기획 의사결정
- MD 팀 — 컬렉션별 성과로 상품 기획 방향 결정

---

# 3. 이 화면을 쓰는 사용자

| MD | 소진율, 소진예상일, 판매 로스, 입고 일정, 컬렉션 순위 |

## 5.4 주문 인사이트 Area

> **데이터 출처**: OMS 주문 데이터 기반 실집계 (NUF_Official_KR 채널)

### Purpose
이커머스 팀이 주문 패턴을 분석해 MD, 마케팅 팀과 공유할 인사이트를 도출하는 영역이다.

### 구성

**베스트셀러 TOP 10** (좌측)
| 컬럼 | 설명 |
| --- | --- |
| 순위 | 주문 건수 기준 |
| 상품명 | 제품명 + 미니 바 차트 |
| 주문 건수 | 선택 기간 주문 수 |
| 취소 건수 | 취소된 주문 수 |
| 취소율 | 취소 건수 / 전체 주문 × 100 |

- 취소율 30% 초과 → 빨간 강조 (MD 이슈 시그널)
- 취소율 20~30% → 주황 강조

**기프트 vs 일반 주문** (우측 상단)
- GIFT / NORMAL 주문 건수 비율
- 수평 바 차트로 비율 시각화
- 기프트 비중이 높은 기간 → 마케팅 선물 캠페인 효과 측정용

**취소 원인 분석** (우측 하단)
- 취소 원인별 비중 (단순 변심, 재고 부족, 배송 지연, 가격 문제 등)
- 원인별 취소 건수 + 수평 바
- 재고 부족 항목은 빨간 강조 → MD 발주 우선순위 시그널

### Display Rules
- "실데이터 기반" 뱃지 표시 — OMS CSV 집계 기준

## 5.5 트래픽 & 전환 분석 Area

> 데이터 출처: GA4 Data API (실데이터 연동 필요 · 현재 더미)
> 기준: 주차(CW) 단위 · 런칭(W36) ~ 현재

### Purpose
전환 퍼널로 이탈 단계를 파악하고, 유입 채널별 트래픽·전환율을 비교하여 마케팅 집행 효율을 판단한다.

### 구성

**전환 퍼널** (전체 너비, 최상단)

방문 → 상품 조회 → 장바구니 → 구매 4단계 시각화

| 단계 | 설명 | 데이터 |
| --- | --- | --- |
| 방문 | 총 세션 수 | GA4 sessions |
| 상품 조회 | 상품 상세 페이지(PDP) 진입 세션 | GA4 productDetailViews |
| 장바구니 | 장바구니 추가 세션 | GA4 add_to_cart events |
| 구매 완료 | 주문 완료 건수 | GA4 purchase events |

- 각 단계 이탈률 표시 (전 단계 대비 %)
- 이탈률 80% 이상 → "최대 이탈" 강조 + 빨간 표시
- 단계별 인사이트 텍스트 자동 표시 (예: "상세페이지 CTA 점검 필요")

**유입 경로별 트래픽 차트** (좌측 7/12)

| 채널 | 설명 | 색상 |
| --- | --- | --- |
| Paid Social | 유료 SNS 광고 (인스타그램 등) | #16a34a (초록) |
| Direct | 직접 방문 | #f59e0b (노랑) |
| Search (Naver+Google) | 검색 유입 | #ef4444 (빨강) |
| Organic Social (IG) | 자연 SNS 유입 | #3b82f6 (파랑) |

- 스택 바 차트 (채널별 유입 적층)
- 전체 전환율 보조 Y축 라인 (`#7c3aed`)

**트래픽 품질 테이블** (우측 5/12)

| 컬럼 | 설명 |
| --- | --- |
| 주차 | CW 기준 (예: W36) |
| 트래픽 | 전체 세션 수 |
| 인당 조회 | 세션당 상품 조회수 |
| 전환율 | 전체 구매 전환율 (%) · 1.5% 이상 보라 강조 |

**채널별 전환율 비교** (전체 너비, 하단)

기간 전체 세션 수 기준 가중평균 전환율을 채널별로 비교한다.

| 표시 항목 | 설명 |
| --- | --- |
| 채널명 + 세션 수 | 해당 채널 기간 합산 세션 |
| 전환율 바 | 최대값 대비 비율 시각화 |
| 전환율 수치 | % (소수점 1자리) |
| 전체 평균 대비 차이 | +/- %p 표시 |
| 인사이트 텍스트 | 기준별 자동 표시 |

전환율 색상 기준:
- < 0.8% → 빨강 ("전환 효율 낮음 · 소재/랜딩 점검 필요")
- 0.8~2.0% → 파랑 ("평균 수준")
- ≥ 2.0% → 초록 ("전환 효율 높음 · 예산 확대 검토")

### What it tells the user
- 어느 단계에서 가장 많이 이탈하는가 → 개선 우선순위
- 어떤 채널이 트래픽은 많지만 전환율이 낮은가 → 광고 소재/랜딩 점검
- 전환율이 높은 채널은 어디인가 → 예산 확대 검토

## 5.6 고객 분석 Area

> 데이터 출처: CRM / 회원 시스템 (연동 필요 · 현재 더미)

### Purpose
신규 고객 유입과 재구매 흐름을 파악하고, 고객이 어떤 상품에 관심을 갖고 있는지 확인한다.

### 구성

**재방문 / 재구매 지표** (우측 상단 3개 카드)

| 지표 | 설명 | 색상 기준 |
| --- | --- | --- |
| 30일 재구매율 | 첫 구매 후 30일 내 재구매한 고객 비율 | ≥25% 초록 / 15~25% 주황 / <15% 빨강 |
| 평균 재구매 주기 | 첫 구매와 두 번째 구매 사이 평균 일수 | 참고용 (색상 없음) |
| 1회 구매자 비율 | 전체 구매 고객 중 1회만 구매한 고객 비율 | ≥70% 빨강 ("재구매 유도 캠페인 필요") |

**신규 vs 재구매 도넛** (좌측)
- 신규 고객 / 재구매 고객 비율 Donut
- 총 구매 고객 수 중앙 표시

**장바구니 / 위시리스트 탭 테이블** (우측)

장바구니 탭:
| 컬럼 | 설명 |
| --- | --- |
| 순위 | 장바구니 추가 건수 순 |
| 상품명 | 제품명 |
| 카테고리 | 중분류 |
| 장바구니 추가 | 추가 건수 |
| 구매 전환율 | 장바구니 → 구매 전환율 (%) |

전환율 색상: ≥40% 초록 / <15% 빨강

위시리스트 탭:
| 컬럼 | 설명 |
| --- | --- |
| 순위 | 위시리스트 추가 건수 순 |
| 상품명 | 제품명 |
| 카테고리 | 중분류 |
| 위시리스트 | 추가 건수 |
| 현재고 | 가용 재고 수량 |
| 소진예상일 | D-N 표시, D-7 이내 빨간 강조 |

### What it tells the user
- 30일 재구매율이 낮으면 → 재구매 유도 이메일/쿠폰 캠페인 필요
- 장바구니 전환율 낮은 상품 → 가격·상세페이지 개선 검토
- 위시리스트 많은데 재고 0 → 재입고 우선순위

## 5.7 상품 성과 Area

### 탭 구성
- `전체` / `재발주 필요` / `소진 필요`

### 분류 기준

| 분류 | 조건 | MD 액션 |
| --- | --- | --- |
| **재발주 필요** | 수요 시그널 높음 + 소진예상일 ≤ 14일 | 재발주 수량·입고 일정 확인 |
| **소진 필요** | 소진예상일 ≥ 180일 또는 온라인 판매가 오프라인의 20% 미만 | 프로모션·번들·단종 검토 |
| **일반** | 위 두 조건 해당 없음 | 모니터링 |

> **수요 시그널** = 판매로스(품절로 못 판 수량) + 재입고 알림 신청 수 합산
> 재고가 없어서 발생한 숨겨진 수요를 포함하므로, 단순 판매량보다 재발주 우선순위 판단에 적합하다.

### 탭별 정렬 기준
| 탭 | 정렬 기준 |
| --- | --- |
| 전체 | 매출 높은 순 |
| 재발주 필요 | 수요 시그널 높은 순 |
| 소진 필요 | 소진예상일 먼 순 (재고 문제 심각한 순) |

### 전체 탭 컬럼

| 카테고리/컬렉션 매출 | 매출 시스템 | 연동 필요 |

## 6.2 갱신 주기
- 매일 오전 9시 배치 업데이트

## 6.3 표기 규칙
- 금액: 원화, 천단위 콤마
- 비율: 소수점 1자리 %
- 소진예상일: `D-14`, `D-7`, `D-3` 형태
- 빈 값: `—`

## 6.4 판매 로스 정의
재고 부족으로 판매하지 못한 수량 × 판매가 합계 (기회 손실액)

## 6.5 소진율 계산
```
소진율(%) = 판매량 / (판매량 + 현재고) × 100
소진예상일 = 현재고 / 일평균 판매량
```

## 6.6 채널별 전환율 계산
```
채널별 가중평균 전환율 = Σ(채널 세션 × 채널 전환율) / 채널 총 세션
```

## 📌 현재 확인된 문제점 (운영 현황)

1. **시스템이 분산되어 있어 데이터 통합 조회가 어려움**
    - 온라인/오프라인 데이터를 한 번에 보는 화면 부족
    - SAC에서도 온오프 통합 모델이 없음
2. **대부분 RAW 데이터를 다운받아 엑셀로 수작업 가공**
    - 자동화 부족, 인력 리소스 소모
3. **국가별·국적별 매출 조회가 온라인에서는 사실상 불가**
    - 해외 배송 미지원 때문에 정식 데이터 제공 어려움
4. **판매 추세 분석도 시스템에서 직접 조회 어려워 직접 가공해야 함**
