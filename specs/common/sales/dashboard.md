**경로:** `common/sales/dashboard`

**문서 목적:** PM이 작성하는 정책서를 기반으로 AI가 UI 프로토타입을 생성할 수 있도록 화면 정책을 정의한다.

> **데이터 기준**
>
> - 이 화면은 **온라인 판매 성과 분석** 중심으로 구성된다.
> - OMS(주문 운영)와 달리 "얼마나, 뭐가, 어떻게 팔렸는가"에 초점을 맞춘다.
> - 온라인 데이터 소스: ADM, OMS (주문일 기준) — 매일 오전 9시 배치 갱신
> - 마케팅 채널 데이터: Google Analytics, 광고 시스템 (더미 표시)

> **사용자**
>
> - **이커머스 팀**: 주문 건수, 베스트셀러, 취소 패턴, 기프트 비중, 전환율
> - **마케팅 팀**: 채널별 ROAS, 캠페인 성과, 유입 채널별 매출
> - **MD 팀**: 소진율, 소진예상일, 발주 지표, 재고/입고 현황

# 1. 화면 이름

**Sales Dashboard**

---

# 2. 이 화면의 목적

이커머스·마케팅·MD 조직이 온라인 판매 성과를 한 화면에서 확인하는 원페이지 대시보드다.

**OMS와의 차이**: OMS는 개별 주문의 현재 처리 상태(배송 어디있어?)를 다룬다. 이 화면은 집계된 판매 성과(뭐가 얼마나 팔렸어?)를 다룬다.

**핵심 가치**
- 분산된 시스템(ADM, OMS, GA, 광고)의 데이터를 한 화면에서 통합 확인
- 이커머스 팀 — 어떤 제품이 팔리고/취소되는지 파악
- 마케팅 팀 — 어떤 채널/캠페인이 매출을 만드는지 파악
- MD 팀 — 재고가 얼마나 남았고 언제 소진될지 파악

## 5.6 주문 인사이트 Area

> **데이터 출처**: OMS 주문 데이터 기반 실집계 (NUF_Official_KR 채널)

### Purpose
이커머스 팀이 주문 패턴을 분석해 MD, 마케팅 팀과 공유할 인사이트를 도출하는 영역이다.

### 구성

**베스트셀러 TOP 10** (좌측)
| 컬럼 | 설명 |
| --- | --- |
| 순위 | 주문 건수 기준 |
| 상품명 | 제품명 |
| 주문 건수 | 선택 기간 주문 수 |
| 매출 | 주문 금액 합계 |
| 취소 건수 | 취소된 주문 수 |
| 취소율 | 취소 건수 / 전체 주문 × 100 |

- 취소율 30% 초과 → 빨간 강조 (MD 이슈 시그널)

**기프트 vs 일반 주문** (우측 상단)
- GIFT / NORMAL 주문 건수 비율
- 기프트 비중이 높은 기간 → 마케팅 선물 캠페인 효과 측정용

**취소 원인 분석** (우측 하단)
- 취소 원인별 비중 (단순 변심, 재고 부족, 배송 지연, 가격 문제 등)
- 원인별 취소 건수

### Display Rules
- "실데이터 기반" 뱃지 표시 — OMS CSV 집계 기준
- 취소율이 높은 제품 → MD 알림 필요 시그널

# 6. 공통 데이터 규칙

## 6.1 데이터 소스
| 데이터 | 소스 | 기준 |
| --- | --- | --- |
| 주문/판매 | OMS | 주문일 기준 |
| 재고 | 재고 시스템 | 기준일 스냅샷 |
| 마케팅 채널 | GA + 광고 시스템 | 더미 |
| 고객 분석 | CRM | 더미 |

## 6.2 갱신 주기
- 매일 오전 9시 배치 업데이트

## 6.3 표기 규칙
- 금액: 원화, 천단위 콤마
- 비율: 소수점 1자리 %
- 소진예상일: `D-14`, `D-7`, `D-3` 형태
- 빈 값: `-`

## 6.4 판매 로스 정의
취소/반품으로 환불된 금액의 합계

## 6.5 소진율 계산
```
소진율(%) = 판매량 / (판매량 + 현재고) × 100
소진예상일 = 현재고 / 일평균 판매량
```

## 📌 **현재 확인된 문제점 정리**

1. **시스템이 분산되어 있어 데이터 통합 조회가 어려움**
    - 온라인/오프라인 데이터를 한 번에 보는 화면 부족
    - SAC에서도 온오프 통합 모델이 없음
2. **대부분 RAW 데이터를 다운받아 엑셀로 수작업 가공**
    - 자동화 부족
    - 인력 리소스 소모
3. **국가별·국적별 매출 조회가 온라인에서는 사실상 불가**
    - 해외 배송 미지원 때문에 정식 데이터 제공 어려움
    - 고객 연락처 국가코드로 추정하는 비정형 방식
4. **판매 추세 분석도 시스템에서 직접 조회 어려워 직접 가공해야 함**
