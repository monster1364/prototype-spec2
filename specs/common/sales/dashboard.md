**경로:** `common/sales/dashboard`

**문서 목적:** PM이 작성하는 정책서를 기반으로 AI가 UI 프로토타입을 생성할 수 있도록 화면 정책을 정의한다.

> **데이터 기준**
>
> - 이 화면은 **온라인 판매 데이터 중심**으로 구성된다.
> - 오프라인 데이터는 채널 비교 영역에서 보조 참고용으로만 표시한다.
> - 온라인 데이터 소스: ADM, OMS (주문일 기준)
> - 오프라인 데이터 소스: Cloud POS Office (판매일 기준)

> **사용자**
>
> - **이컴 팀**: 전환율, 재입고 알림, 라인/카테고리 판매 추이 중심
> - **MD 팀**: 판매량, 소진율, 소진예상일, 발주 지표 중심
> - 두 팀이 공통으로 보는 단일 화면으로 설계한다.

# 1. 화면 이름

**Sales Dashboard**

---

# 2. 이 화면의 목적

이컴·MD 조직이 온라인 판매 데이터를 한 화면에서 통합 확인할 수 있는 원페이지 대시보드다.

현재 ADM, OMS, SAC 등 분산된 시스템에서 각각 RAW 데이터를 내려받아 수작업 가공하는 방식을 대체한다.
자동화된 집계 기반으로 판매 현황·재고 흐름·제품 성과를 실시간에 가깝게 제공한다.

**핵심 가치**

- 매일 오전 9시 배치 기준으로 전날까지의 데이터를 자동 집계한다.
- 이컴·MD가 각자 엑셀을 가공하지 않아도 동일한 수치를 볼 수 있어야 한다.
- 제품 테이블 하나에서 판매 성과(이컴)와 재고/소진(MD)을 함께 확인할 수 있어야 한다.

---

# 3. 이 화면을 쓰는 사용자

| 팀 | 주요 관심 지표 |
| --- | --- |
| 이컴 | 제품별 주문 건수, 일자별 주문 추이, 전환율, 재입고 알림신청 수, 라인/카테고리 판매 추이 |
| MD | 일/월 판매량, 판매 로스, 소진율, 소진예상일, 발주 지표, DTC/MTC, 입고 일정 |

# 4. 화면 레이아웃

원페이지 구성. 위에서 아래 순서로 아래 7개 영역을 배치한다.

## 4.1 Page Header Area
- 화면 제목 + 데이터 기준일 (배치 갱신 시점)
- 기간 필터 (일별 / 월별 / 연도별)
- 날짜 범위 선택
- 카테고리 / 라인 필터
- Excel 다운로드 버튼

## 4.2 KPI Summary Area
- 온라인 중심 핵심 지표 6개 카드

## 4.3 채널 비교 + 매출 추이 Area
- 좌: 온라인/오프라인 채널 비중 (오프라인은 보조)
- 우: 온라인 매출 추이 Line 차트 + 전년 동기 비교선

## 4.4 카테고리 / 라인 판매 분석 Area
- 카테고리별 판매 수평 Bar
- 컬렉션/라인별 판매 순위 테이블

## 4.5 제품 성과 Area
- 판매 TOP N 테이블
- 이컴 지표(전환율)와 MD 지표(소진율, 소진예상일)를 동일 테이블에 표시
- 기회/위험 분류 탭

## 4.6 재고/입고 현황 Area ← MD 핵심
- 소진 임박 상품 목록
- 입고 예정 상품 목록
- DTC / MTC 요약

## 4.7 고객 분석 Area
- 신규 vs 재구매 비율
- 장바구니 / 위시리스트 추가 수

# 5. 영역별 상세 명세

## 5.1 Page Header Area

### Displayed Data

- 페이지 제목: `Sales Dashboard`
- 데이터 기준: `기준: YYYY-MM-DD 09:00 배치 · 온라인(주문일 기준)`

- 기간 토글 변경 → 날짜 범위 단위 변경 + 전체 데이터 갱신
- 필터 변경 → 전체 영역 연동 갱신

## 5.5 제품 성과 Area

### Purpose
이컴·MD가 동일 테이블에서 판매 성과와 재고 흐름을 함께 확인한다.

### 탭 구성
- `전체` / `기회 상품` / `위험 상품`

### 테이블 컬럼

| 컬럼 | 설명 | 주 사용 팀 |
| --- | --- | --- |
| 순위 | 판매량 기준 | 공통 |
| 상품명 | 제품명 | 공통 |
| 카테고리 | 제품 카테고리 | 공통 |
| 판매량 | 선택 기간 판매 수량 | 공통 |
| 판매 금액 | 선택 기간 판매 금액 | 공통 |
| 일평균 판매량 | 판매량 / 기간(일) | MD |
| 전환율 | 상품 페이지 방문 대비 구매 비율 | 이컴 |
| 현재고 | 현재 가용 재고 수량 | MD |
| 소진율 | 판매량 / (판매량 + 현재고) × 100 | MD |
| 소진예상일 | 현재고 / 일평균 판매량 (일 단위) | MD |
| 분류 | 기회 / 위험 / 일반 뱃지 | 공통 |

### 기회/위험 분류 기준
- **기회**: 최근 7일 판매 속도 > 이전 7일 × 1.2, 또는 전환율 > 평균 × 1.2
- **위험**: 최근 7일 판매량 = 0, 또는 소진율 < 10% (재고 적체)

# 6. 공통 데이터 규칙

## 6.1 데이터 소스 및 기준

| 데이터 | 소스 | 날짜 기준 |
| --- | --- | --- |
| 온라인 판매 | ADM, OMS | **주문일 기준** |
| 오프라인 판매 | Cloud POS Office | 판매일 기준 (보조) |
| 재고 | 재고 시스템 | 기준일 스냅샷 |

- 온라인/오프라인 날짜 기준이 다름을 화면에 명시한다.
- 이 화면은 온라인(주문일 기준) 데이터를 primary로 사용한다.

## 6.2 데이터 갱신 주기
- **매일 오전 9시** 배치 업데이트
- 실시간 갱신 없음 — 기준 시점 항상 표시

## 6.3 기간 단위
- 일별: 최대 90일
- 월별: 최대 24개월
- 연도별: 최대 5년

## 6.4 표기 규칙
- 금액: 원화, 천단위 콤마 (예: 12,500,000원)
- 비율: 소수점 1자리 % (예: 23.4%)
- 증감: ▲ / ▼ + 숫자 + % (예: ▲12.3%)
- 소진예상일: `D-14`, `D-7`, `D-3` 형태
- 빈 값: `-`

## 6.5 판매 로스 정의
- 취소된 주문 금액 + 반품으로 환불된 금액의 합계
- 소진율과 함께 MD의 핵심 손실 지표

## 6.6 소진율 계산
```
소진율(%) = 판매량 / (판매량 + 현재고) × 100
소진예상일 = 현재고 / 일평균 판매량
```

## 6.7 Excel 다운로드
- 파일명: `sales_YYYYMMDD_YYYYMMDD.xlsx`
- 현재 필터 기준 전체 데이터 포함

# 9. AI에게 원하는 산출물

## 9.1 프로토타입 범위
- Desktop 기준 원페이지
- 상단 필터 + KPI 6개 + 채널/추이 차트 + 카테고리 분석 + 제품 성과 + 재고/입고 현황 + 고객 분석

## 9.2 반드시 반영할 것
- 온라인 데이터 primary, 오프라인 보조 처리
- 제품 테이블에 소진율/소진예상일/현재고 컬럼 포함
- 재고/입고 현황 섹션 (소진 임박 + 입고 예정 + DTC/MTC)
- KPI에 판매 로스, 재입고 알림신청 수 포함
- 데이터 기준(온라인, 주문일 기준) 화면 내 명시

## 9.3 Mock 데이터 추가/변경 항목

**기존 타입 변경**
- `ProductPerformanceItem`: `currentStock`, `sellThroughRate`, `daysToSellOut` 추가

**신규 타입**
- `SalesInventoryItem`: 소진 임박 상품 (상품명, 현재고, 일평균 판매량, 소진예상일, 알림신청 수)
- `InboundItem`: 입고 예정 상품 (상품명, 입고예정일, 입고예정수량, 입고율, 상태)
- `DtcMtcSummary`: DTC/MTC 요약 (DTC 수량/금액, MTC 수량/금액)

**기존 KPI 변경**
- `SalesDashboardKPI`의 `conversionRate` 유지
- `salesLoss` (판매 로스 금액), `salesLossDelta` 추가
- `restockAlertCount` (재입고 알림신청 수), `restockAlertDelta` 추가
- `newCustomers`, `repeatRatio` 제거 → 5.7 고객 분석 영역으로 이동
