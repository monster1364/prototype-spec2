# 1. 화면 이름

**Sales Dashboard**

---

# 2. 이 화면의 목적

**이커머스·MD 조직**이 온라인 판매 성과를 한 화면에서 확인하는 원페이지 대시보드다.

이 화면은 집계된 판매 성과(뭐가 얼마나 팔렸어?)를 다룬다.

**핵심 가치**
- 이커머스 팀 — 어떤 제품이 팔리고/취소되는지 파악
- MD 팀 — 재고 소진 속도와 입고 일정 파악, 다음 발주/기획 의사결정
- MD 팀 — 어떤 채널에서 어떤 상품이 팔리는지 파악해 상품 기획 방향 결정

---

| MD | 소진율, 소진예상일, 판매 로스, 입고 일정 |

## 5.5 상품 성과 Area

### 탭 구성
- `전체` / `재발주 필요` / `소진 필요`

### 분류 기준

| 분류 | 조건 | MD 액션 |
| --- | --- | --- |
| **재발주 필요** | 수요 시그널 높음 + 소진예상일 ≤ 14일 | 재발주 수량·입고 일정 확인 |
| **소진 필요** | 소진예상일 ≥ 180일 또는 온라인 판매가 오프라인의 20% 미만 | 프로모션·번들·단종 검토 |
| **일반** | 위 두 조건 해당 없음 | 모니터링 |

> **수요 시그널** = 판매로스(품절로 못 판 수량) + 재입고 알림 신청 수 합산
> 재고가 없어서 발생한 숨겨진 수요를 포함하므로, 단순 판매량보다 재발주 우선순위 판단에 적합하다.

### 탭별 정렬 기준
| 탭 | 정렬 기준 |
| --- | --- |
| 전체 | 매출 높은 순 |
| 재발주 필요 | 수요 시그널 높은 순 |
| 소진 필요 | 소진예상일 먼 순 (재고 문제 심각한 순) |

### 전체 탭 컬럼
| 컬럼 | 설명 | 팀 |
| --- | --- | --- |
| 순위 | 매출 기준 전체 순위 | 공통 |
| 상품명 | 제품명 | 공통 |
| 카테고리 | 중분류 | 공통 |
| 판매량 | 선택 기간 판매 수량 | 공통 |
| 판매 금액 | 선택 기간 판매 금액 | 공통 |
| 판매 속도 | 일평균 판매 수량 | MD |
| 전환율 | 구매 완료 / 상품 조회 수 | 이커머스 |
| 현재고 | 가용 재고 수량 | MD |
| 소진율 | 판매량 / (판매량 + 현재고) × 100 | MD |
| 소진예상일 | 현재고 / 일평균 판매량 (D-N) | MD |
| 분류 | 재발주·소진필요·일반 | 공통 |

### 재발주 필요 탭 컬럼
| 컬럼 | 설명 |
| --- | --- |
| 판매 속도 | 일평균 판매 수량 (초록 강조) |
| 현재고 | 30개 이하 빨간 강조 |
| 소진예상일 | D-N 표시, D-7 이내 빨간 / D-3 이내 "품절임박" 배지 |
| **수요 시그널** | 판매로스 + 재입고 알림 신청 수 합산 — 재발주 우선순위 기준 |
| 판매 금액 | 참고용 |

### 소진 필요 탭 컬럼
| 컬럼 | 설명 |
| --- | --- |
| 현재고 | 빨간 강조 (재고 과다) |
| 판매 속도 | 일평균 판매 수량 |
| **소진예상일** | 실제 날짜(YYYY-MM-DD)로 표시 |
| 온라인 판매 | 선택 기간 온라인 판매 수량 |
| 오프라인 판매 | 선택 기간 오프라인 판매 수량 |
| **온/오프 갭** | 온라인 / 오프라인 × 100 — 20% 미만이면 빨간 칩 표시 |

## 5.6 주문 인사이트 Area

> **데이터 출처**: OMS 주문 데이터 기반 실집계 (NUF_Official_KR 채널)

### Purpose
이커머스 팀이 주문 패턴을 분석해 MD, 마케팅 팀과 공유할 인사이트를 도출하는 영역이다.

### 구성

**베스트셀러 TOP 10** (좌측)
| 컬럼 | 설명 |
| --- | --- |
| 순위 | 주문 건수 기준 |
| 상품명 | 제품명 |
| 주문 건수 | 선택 기간 주문 수 |
| 매출 | 주문 금액 합계 |
| 취소 건수 | 취소된 주문 수 |
| 취소율 | 취소 건수 / 전체 주문 × 100 |

- 취소율 30% 초과 → 빨간 강조 (MD 이슈 시그널)

**기프트 vs 일반 주문** (우측 상단)
- GIFT / NORMAL 주문 건수 비율
- 기프트 비중이 높은 기간 → 마케팅 선물 캠페인 효과 측정용

**취소 원인 분석** (우측 하단)
- 취소 원인별 비중 (단순 변심, 재고 부족, 배송 지연, 가격 문제 등)
- 원인별 취소 건수

### Display Rules
- "실데이터 기반" 뱃지 표시 — OMS CSV 집계 기준
- 취소율이 높은 제품 → MD 알림 필요 시그널

## 5.9 트래픽 & 전환 분석 Area

> 데이터 출처: GA4 Data API (실데이터 연동 필요 · 현재 더미)
> 기준: 주차(CW) 단위 · 런칭(W36) ~ 현재

### Purpose
유입 경로(채널)별 세션 수와 구매 전환율을 주차 단위로 파악하여 마케팅 집행 효율을 확인한다.

### Displayed Data
**좌: 유입 경로별 트래픽 차트 (스택 바 + 전환율 라인)**

| 채널 | 설명 | 색상 |
| --- | --- | --- |
| Paid Social | 유료 SNS 광고 (인스타그램 등) | #16a34a (초록) |
| Direct | 직접 방문 | #f59e0b (노랑) |
| Search (Naver+Google) | 검색 유입 | #ef4444 (빨강) |
| Organic Social (IG) | 자연 SNS 유입 | #3b82f6 (파랑) |

- 전환율 라인: 보조 Y축, 보라색 점선 (`#7c3aed`)

**우: 트래픽 품질 테이블**

| 컬럼 | 설명 |
| --- | --- |
| 주차 | CW 기준 (예: W36) |
| 트래픽 | 전체 세션 수 |
| 인당 조회 | 세션당 상품 조회수 |
| 전환율 | 구매 전환율 (%) · 1.5% 이상 강조 표시 |

### Display Rules
- 전환율 ≥ 1.5%: 보라색 Bold 표시
- 차트 Y축: 0K ~ max × 1.1 단위 표시
- 전환율 라인: 최소·최대값 ±20% 여백으로 Y축 자동 조정

### What it tells the user
- 어느 주차에 트래픽이 집중됐는지
- Paid Social 중단 후 자연 유입 비중 변화
- 전환율이 높은 주차의 채널 구성 패턴 → 다음 캠페인 기준

### Interaction
- 현재: 정적 표시 (GA4 연동 후 기간 필터 연동 예정)

## 6.4 판매 로스 정의
취소/반품으로 환불된 금액의 합계

## 6.5 소진율 계산
```
소진율(%) = 판매량 / (판매량 + 현재고) × 100
소진예상일 = 현재고 / 일평균 판매량
```

## 📌 **현재 확인된 문제점 정리**

1. **시스템이 분산되어 있어 데이터 통합 조회가 어려움**
    - 온라인/오프라인 데이터를 한 번에 보는 화면 부족
    - SAC에서도 온오프 통합 모델이 없음
2. **대부분 RAW 데이터를 다운받아 엑셀로 수작업 가공**
    - 자동화 부족
    - 인력 리소스 소모
3. **국가별·국적별 매출 조회가 온라인에서는 사실상 불가**
    - 해외 배송 미지원 때문에 정식 데이터 제공 어려움
    - 고객 연락처 국가코드로 추정하는 비정형 방식
4. **판매 추세 분석도 시스템에서 직접 조회 어려워 직접 가공해야 함**
